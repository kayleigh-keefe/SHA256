# SHA-256 Implementation

A C++ implementation of the SHA-256 hashing algorithm.

## Features
- Compliant with [FIPS 180-4](https://nvlpubs.nist.gov/nistpubs/fips/nist.fips.180-4.pdf).
  - Key Points of FIPS 180-4 (Federal Information Processing Standard 180-4)
    - For SHA-256, when a message is is any length less than 2^64 bits is input,the resulting output is a message digest.
    - The message digest is always exactly 256 bits in length.
    - The hash functions defined herein, including SHA-256, are called secure because it is computationally infeasible to
      1. find a message that corresponds with a given digest
      2. find two different messages that produce the same message digest.
    - The algorithm is one-way.
    - Such an algorithm is useful in the verification of digital signatures and verifying message integrity.
    - Can be described in two stages:
      1. preprocessing
      2. hash computation
        - generates a message schedule from the padded message which and uses that schedule, along with functions, constants, and word operations to iteratively generate a series of hash values. The final hash value generated by the hash computation is used to determine the message digest.
    - SHA-256 message size (bits): < 2^64; Block Size (bits): 512; Word Size (bits): 32; Message Digest Size (bits): 256
    - *Big Endian* is used throughout this specification to represent bit order within each 32-bit word.
    - Utilizes the fractional parts of the first 64 prime number cubed roots (see Sha256.h search 'k' for details).
    - Hash initial calues are the first 32 bits of the decimal portions of the square roots of the first 8 prime numbers (see Sha256.h search for h0 through h7 for details).
- Handles multi-block message processing.
- Includes a test suite with known test vectors.
